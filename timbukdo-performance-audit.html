Severe bundle bloat detected

The build ships ~18 MB of JavaScript for a mobile webview.

Most of it is illustration SVGs compiled into React components. Fixing that one pattern recovers the majority of the budget — everything else is cleanup.

C1 Giant SVG icons imported as React components ~12 MB JS ›

Multi-megabyte illustration/banner SVGs are converted to inline React components via ?react. A single file like Tim_Rating_short.svg (2.4 MB raw) becomes a 2.46 MB JS file — svgr turns every <path> into a React.createElement() call, inflating size 20–50% over the raw SVG.

C2 activity.png is 1.94 MB, shipped verbatim 1.94 MB ›

A 1.94 MB PNG is copied as-is into the build. Convert to WebP at a reduced resolution.

npx sharp-cli -i src/assets/icons/activity.png \
  -o src/assets/icons/activity.webp --webp --quality 80 --resize 800
↓ 1.94 MB → ~80 KB
C3 india-composite.json eagerly loaded in Locate 112 KB ›

GeoJSON is statically bundled into the Locate chunk even when the map is never shown. Load it dynamically, or better, from a CDN/API endpoint.

// Locate.tsx
// ❌ bundled even if map is never rendered
import indiaBoundary from "@/assets/india-composite.json";

// ✅ loaded only when the map renders
const indiaBoundary = await import("@/assets/india-composite.json")
  .then(m => m.default);

H3Firebase SDK loaded eagerly at startup100–150 KB›

Auth and Analytics both initialize at module load in firebase.ts, regardless of route. Auth is only needed on Login/OTP; Analytics can be deferred entirely.

// ✅ lazy init
export const getFirebaseAuth = async () => {
  if (!_auth) {
    const { getAuth } = await import("firebase/auth");
    _auth = getAuth(app);
  }
  return _auth;
};
H4useNativeEventTracker chunk is 213 KB213 KB›

Investigate what's pulled into this chunk — likely Firebase Analytics is bundled in transitively via this utility.

H5Three overlapping icon systems30–50 KB›

M2RTK Query placeholder — dead code›

api.ts registers an RTK Query slice pointed at jsonplaceholder.typicode.com with zero endpoints, still adding middleware overhead to the store. Remove until actually needed.

M3Missing React.memo on frequently re-rendered components›

Only CaResult.tsx uses React.memo today. GigCard, DealCard, FooterTabs, ProfileItem, and BannerCarousel all re-render unnecessarily in lists or on parent updates.

M4Locate.tsx — 509-line monolith›

Contains inline MapController and NoVendors components (the latter re-created every render), plus Haversine math, pincode geocoding, native bridge calls, and data fetching all in one file. Extract each concern into its own hook/component.

M5Leaflet not dynamically imported285 KB›

Leaflet + CSS + GeoJSON sit in the already-lazy Locate chunk, but the map itself only needs to load once a user toggles to map view — defer it a level further with its own lazy() boundary.

M6Duplicate MascotBanner import›

Tim_Main_banner.svg?react is imported in both EarnView.tsx and Shop.tsx. Resolves itself once C1's WebP conversion lands.

M7Tailwind config references uninstalled plugins›

tailwind.config.js requires @tailwindcss/forms, typography, aspect-ratio, and line-clamp — none are in package.json. Since the project runs Tailwind v4 via @tailwindcss/vite, this whole config file may be silently ignored; verify it's actually consumed.

Low 7 issues
L1message.config() called on every render›

In App.tsx (line 139), move the call into a useEffect(() => {...}, []).

L2waitForBridge() duplicated across files›

Copy-pasted in HomePage.tsx and Locate.tsx. Extract to a shared src/utils/nativeBridge.ts.

L3Bundle visualizer plugin always active›

visualizer({ open: true }) opens stats.html on every build. Gate it behind process.env.ANALYZE.

L4Redundant user-data API calls›

ProtectedApp.tsx and HomePage.tsx each fetch user data independently. Consolidate into one Redux fetch, consume via useAppSelector.

L5Facebook Pixel script blocks rendering›

Inline synchronous script in index.html. Defer until after window load.

L6No gzip/brotli compression configured›

Add vite-plugin-compression for both gzip and brotli output.

L7No manual chunk strategy›

Split vendor code predictably: vendor-react, vendor-antd, vendor-redux, vendor-firebase, vendor-motion.
